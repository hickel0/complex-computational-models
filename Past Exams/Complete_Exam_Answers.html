EXAM 1: 2020/2021 (CA4024)

QUESTION 1: Predator-Prey Population Dynamics [25 Marks]

Q1(a) [4 Marks] - Determine coefficients a, b, c, d

Given Information:

  • Owl population decays by 50% annually without rats
  • Owls convert rat kills to population growth with 40% efficiency
  • Rat population grows by 10% annually without owls
  • Each owl reduces rat population by 0.1

Solution:

For the rat population equation: Rn+1 = aRn + bOn

  • In absence of owls: rats grow by 10%, so a = 1.1
  • Each owl reduces rats by 0.1, so b = -0.1

For the owl population equation: On+1 = cRn + dOn

  • In absence of rats: owls decay by 50%, so d = 0.5
  • Owls convert rat kills (0.1 rats per owl) with 40% efficiency: c = 0.1 x 0.4 = 0.04
Final Answer:
a = 1.1, b = -0.1, c = 0.04, d = 0.5

Q1(b) [11 Marks] - Matrix System, Eigenvalues, Long-term Survival

Matrix Form:

A = [1.1   -0.1]
    [0.04   0.5]

Finding Eigenvalues:

det(A - λI) = 0
(1.1-λ)(0.5-λ) - (-0.1)(0.04) = 0
λ² - 1.6λ + 0.554 = 0

λ = (1.6 ± √0.344)/2
λ₁ = 1.093 (dominant eigenvalue)
λ₂ = 0.507

Eigenvector for λ₁ = 1.093:

(A - λ₁I)v = 0
0.007v₁ - 0.1v₂ = 0
v₁ = 14.3v₂

Eigenvector: [14.3, 1]
Long-term Survival Analysis:
Since λmax = 1.093 > 1, both populations will grow continuously over time.
The ratio of rats to owls will stabilize at approximately 14.3:1.
The rat population survives and grows in the long term.

Q1(c) [10 Marks] - 50% More Predation, Critical Rate

New Scenario: b changes from -0.1 to -0.15 (50% more rats killed per owl)

New matrix with updated values:

A' = [1.1   -0.15]
     [0.06   0.5 ]

New eigenvalue λmax = 1.085 > 1, so populations still grow.

Critical Rate of Predation:

Set λmax = 1 for stability:

Let b = -k and c = 0.4k
det(A - I) = 0
(0.1)(-0.5) + 0.4k² = 0
k² = 0.125
k = 0.354
Critical predation rate: |b| = 0.354 rats per owl per year
At this rate, populations remain stable. Above this rate, both populations decline.

QUESTION 2: Time Series Decomposition [25 Marks]

Q2(a) [4 Marks] - Two Decomposition Models

1. Additive Model:

Yt = Tt + St + Ct + It

When to use: When seasonal fluctuations are roughly constant regardless of the trend level

2. Multiplicative Model:

Yt = Tt × St × Ct × It

When to use: When seasonal fluctuations scale proportionally with the trend level

Q2(b) [6 Marks] - Definitions

Detrending: The process of removing the trend component from a time series to isolate other components. Calculate the trend using moving averages and subtract (additive) or divide (multiplicative) from original data.

Seasonal Index: A numerical value representing the typical effect of a particular season/period relative to the average. Average the detrended values for each season across years.

Seasonally Adjusted: Data with the seasonal component removed, revealing underlying trend and irregular movements. For additive: Yt - St. Allows fair comparison across different seasons.

QUESTION 3: Stationarity and ARIMA [25 Marks]

Q3(a) [6 Marks] - Trend Stationary vs Difference Stationary

Trend Stationary:

  • Has a deterministic (predictable) trend
  • Underlying process is stationary once trend is subtracted
  • Example: Yt = α + βt + εt
  • How to render stationary: Fit regression line and subtract: Zt = Yt - (α + βt)

Difference Stationary:

  • Has a stochastic (random walk) trend
  • Shocks are permanent; series never returns to previous level
  • Example: Yt = Yt-1 + εt
  • How to render stationary: Take first differences: ΔYt = Yt - Yt-1

Q3(b) [11 Marks] - ACF, PACF, and Information Criteria

ACF (Auto-Correlation Function): Measures correlation between a time series and its lagged values, including both direct and indirect effects. Useful for identifying MA(q) order.

PACF (Partial Auto-Correlation Function): Measures correlation between Yt and Yt-k after removing effects of intermediate lags. Useful for identifying AR(p) order.

ModelACF PatternPACF Pattern
AR(p)Tails off graduallyCuts off after lag p
MA(q)Cuts off after lag qTails off gradually
ARMA(p,q)Tails offTails off

Information Criteria:

  • AIC = -2ln(L) + 2k
  • BIC = -2ln(L) + k·ln(n)
  • Select model with lowest AIC/BIC value

Q3(c) [8 Marks] - Identifying Models from ACF/PACF

Case (i): ACF cuts off after lag 1, PACF tails off → MA(1) or ARIMA(0,0,1)

Case (ii): ACF shows spikes at lags 1, 12, 24; PACF spikes at lags 1 and 12 → SARIMA(1,0,0)(1,0,0)₁₂

QUESTION 4: Complex Systems [25 Marks]

Q4(a) [3 Marks] - Define Complex System

A Complex System is a system composed of many interacting components (agents) where:
  1. Emergence: Collective behavior cannot be predicted from individual components
  2. Non-linearity: Small changes can lead to disproportionately large effects
  3. Adaptation: Components may change behavior based on feedback
  4. Self-organization: Global patterns arise without central control
Examples: Ant colonies, financial markets, weather systems, the human brain

Q4(b) [10 Marks] - Emergent Properties vs Self-Organization

Emergent Properties:

  • Macro-level properties arising from micro-level interactions
  • Example: Flocking behavior in birds - individual birds follow simple rules (separation, alignment, cohesion), but the flock exhibits complex, coordinated patterns

Self-Organization:

  • Spontaneous formation of ordered structures without external direction
  • Example: Crystal formation - molecules arrange into regular lattices based on local energy minimization
AspectEmergenceSelf-Organization
FocusProperties/behaviorsProcess/mechanism
WhatThe "what" (result)The "how" (process)
Can self-organization exist without emergence?
Yes. Crystallization is a classic example - molecules self-organize into regular structures through local energy minimization, but this pattern is predictable from molecular properties.

Q4(c) [12 Marks] - Steps in Modeling a Complex System

Example: Traffic Flow Simulation

  1. Define Purpose and Boundaries: Understand how traffic jams form
  2. Identify Agents and Environment: Vehicles (agents), Road network (environment)
  3. Define Agent Properties: Speed, destination, driver behavior
  4. Define Interaction Rules: Maintain safe distance, lane changing decisions
  5. Implement Computational Model: Code behaviors in simulation framework
  6. Validate and Calibrate: Compare to real traffic data
  7. Analyze Emergent Behaviors: Observe phantom jams, shockwaves

Q5(c) [9 Marks] - Conway's Game of Life and Emergence

Rules (using Moore neighborhood):

  1. Birth: Dead cell with exactly 3 live neighbors becomes alive
  2. Survival: Live cell with 2-3 live neighbors survives
  3. Death by underpopulation: Live cell with <2 neighbors dies
  4. Death by overpopulation: Live cell with >3 neighbors dies

Emergent Properties:

  • Still Lifes: Stable configurations (Block, Beehive)
  • Oscillators: Repeating patterns (Blinker, Toad)
  • Gliders: Patterns that move across the grid
  • Turing Completeness: Can simulate any computer

QUESTION 6: Dynamical Systems and Chaos [25 Marks]

Q6(a) [9 Marks] - Attractors, Strange Attractors, and Limit Cycles

Point Attractor: System converges to a single equilibrium value (e.g., damped pendulum)

Limit Cycle: System settles into a periodic, repeating trajectory (e.g., heartbeat, predator-prey oscillations)

Strange Attractor: Fractal structure with chaotic, non-repeating trajectories sensitive to initial conditions (e.g., Lorenz attractor)

Q6(b) [8 Marks] - Logistic Map Bifurcation Diagram

Logistic Map: xn+1 = r·xn(1 - xn)
  • Point A (r ≈ 3.45): First period-doubling bifurcation - single stable point splits into period-2 cycle
  • Point B (r ≈ 3.54): Second period-doubling - system oscillates through 4 values
  • Point C (r ≈ 3.83): Window of stability (Period-3) within chaotic region
  • Point D (r ≈ 3.9+): Fully chaotic region - dense, apparently random scatter

Q6(c) [8 Marks] - Lotka-Volterra Limit Cycle

Cycle Description:

  1. Phase 1: Prey abundant → Predators increase
  2. Phase 2: Predators peak → Heavy predation → Prey decline
  3. Phase 3: Prey scarce → Predators starve and decline
  4. Phase 4: Predators scarce → Prey recover → Return to Phase 1

This creates a perpetual closed-loop trajectory in phase space.

QUESTION 1: Time Series Forecasting [25 Marks]

Q1(a) [4 Marks] - Ex Post vs Ex Ante Forecasting

Ex Post Forecasting:

  • Forecasts made for periods where actual values are already known
  • Used for model validation, backtesting, and error calculation
  • Considered the "gold standard" because accuracy can be objectively measured

Ex Ante Forecasting:

  • Forecasts predicting genuinely unknown future values
  • True forecasting with no actual values for comparison (yet)
  • Higher risk: model errors only discovered after the fact

Characteristics of a Good Model:

  1. Parsimony: Simple as possible while capturing essential dynamics
  2. Validity: Produces results consistent with real-world observations

QUESTION 4: ABM and CA [25 Marks]

Q4(a) - ABM Agent Properties and Boids

Typical Properties of ABM Agents:

  1. Autonomy: Independent decisions
  2. Heterogeneity: Different attributes and behaviors
  3. Bounded Rationality: Limited information
  4. Adaptation: Learning and changing behavior
  5. Mobility: Can move through space
  6. Internal State: Memory and variables
  7. Goal-Directed: Pursue objectives

Boids Flocking Rules:

  • Separation: Steer away from nearby birds to avoid collision
  • Alignment: Match direction and speed of neighbors
  • Cohesion: Steer toward average position of nearby birds

Q4(b) - Louvain Method

Algorithm Steps:

  1. Phase 1: Start with each node in its own community
  2. Calculate modularity gain from moving each node to neighbor's community
  3. Move nodes to communities giving maximum positive gain
  4. Phase 2: Build new network where nodes are communities
  5. Apply Phase 1 again to aggregated network
  6. Repeat until modularity no longer increases

Q4(d) - Forest Fire CA Model

States:

  • 0 = Empty (burned out)
  • 1 = Tree (alive, burnable)
  • 2 = Burning (currently on fire)

Transition Rules:

  • Empty → Tree: With probability p (regrowth)
  • Tree → Burning: If neighbor is Burning OR lightning strikes
  • Burning → Empty: Always (fire consumes in one step)

Different Dimensions:

  • 1-D: Fire can only spread left/right; any gap blocks fire
  • 2-D: Fire has multiple paths; harder to contain
  • 3-D: Exponentially more paths; extremely difficult containment

QUESTION 5: Markov Processes and MCMC [25 Marks]

Q5(a) [4 Marks] - Markov Process Definition

A Markov Process is a stochastic process where the probability of the next state depends ONLY on the current state, not on the sequence of states that preceded it.

P(Xt+1 | Xt, Xt-1, ...) = P(Xt+1 | Xt)

Main Assumptions:
  1. Memorylessness (Markov Property)
  2. Finite State Space
  3. Time Homogeneity
  4. Stationarity

Q5(c)-(d) - MCMC

MCMC (Markov Chain Monte Carlo): A class of algorithms that sample from probability distributions by constructing a Markov chain whose equilibrium distribution equals the target distribution.

When to Use: Bayesian inference, high-dimensional integration, complex probability models

Metropolis-Hastings Sampler:

  1. Propose θ' from proposal distribution
  2. Calculate acceptance ratio: α = min(1, P(θ'|D) / P(θcurrent|D))
  3. Accept with probability α, else stay

QUESTION 1: Time Series Types and ARIMA [25 Marks]

Q1(a) [9 Marks] - Continuous vs Discrete Time Series

Continuous Time Series:

  • Values can change at any point in time
  • Examples: Temperature readings, stock prices during trading, blood pressure

Discrete Time Series:

  • Values recorded at specific, equally-spaced intervals
  • Examples: Daily closing prices, monthly unemployment, quarterly GDP

How Discrete Series Arise:

  1. Sampling continuous phenomena at intervals
  2. Aggregation over periods
  3. Inherently discrete events
  4. Data collection constraints

QUESTION 3: Forecasting Methods [25 Marks]

Q3(a) [6 Marks] - Qualitative vs Quantitative

Qualitative: Based on expert judgment, intuition (Delphi method, surveys)

Quantitative: Based on mathematical analysis of historical data (ARIMA, regression)

Q3(c) [9 Marks] - Naïve Model and Exponential Smoothing

Naïve Model: ŷt+1 = yt

Simple Exponential Smoothing: ŷt+1 = αyt + (1-α)ŷt

Reduction to Naïve: When α = 1:

ŷt+1 = 1·yt + 0·ŷt = yt

This is exactly the Naïve model. Naïve forecasting is a special case of Exponential Smoothing with α = 1.

QUESTION 6: Network Analysis [25 Marks]

Q6(a) [4 Marks] - Community Detection

Community Detection: Identifying groups of nodes more densely connected internally than with the rest of the network.

Usefulness: In social networks - identify friend groups, interest communities, customer segments.

Q6(b)-(c) - Adjacency Matrix and Modularity

When each node is its own community, modularity Q = 0 or negative (no within-community edges contribute positively).
